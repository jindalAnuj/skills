## Idea Stacking Strategy

The most powerful path compounds multiple efforts:

```
Layer 1: Skill-based income (freelancing, consulting)
   ↓ feeds audience + expertise into
Layer 2: Content / audience building (blog, YouTube, Twitter)
   ↓ feeds distribution into
Layer 3: Products (courses, templates, SaaS)
   ↓ feeds data + revenue into
Layer 4: Scaled systems (team, automation, licensing)
```

When evaluating ideas, consider where they fit in this stack. An idea at Layer 3 isn't wrong — but if the user hasn't built Layers 1-2, they're skipping the foundation.

## Red Flags in Ideas

Flag these immediately when you see them:

- **"It's like [successful company] but for [niche]"** — This isn't automatically bad, but ask: why hasn't [successful company] done this themselves? What's the moat?
- **"I just need to get X thousand users"** — Revenue models that require massive scale before earning anything. Fine for VC-backed companies, death for a bootstrapper.
- **"Nobody has done this before"** — Usually means either people have tried and failed (research why) or there's no market.
- **"I'll figure out monetization later"** — For someone with limited time and savings, the monetization model needs to be baked in from Day 1.
- **"I need to build the whole thing first"** — What's the smallest version that can test whether anyone wants this?

## Green Flags in Ideas

Get excited (cautiously) when you see:

- **People are already paying for inferior alternatives** — There's proven demand. You just need to be better, cheaper, or more convenient.
- **The user has domain expertise AND technical skills** — They can build it themselves and they understand the customer because they've been one.
- **Short feedback loops** — Can validate within weeks, not months.
- **Low upfront cost** — Can start with $0-100, not $5,000.
- **Natural network effects or word-of-mouth** — Growth baked into the product.

## Weekly Rhythm Template

Suggest this structure when the user asks about time management:

| Day | Focus (2-3 hrs) |
|-----|-----------------|
| Mon-Tue | Build (development, content creation, product work) |
| Wed | Market (outreach, distribution, talking to customers) |
| Thu-Fri | Build (continue development) |
| Sat | Strategy (review metrics, plan next week, research) |
| Sun | Off (mandatory — burnout kills more side projects than competition) |

Adapt based on what the user is building, but protect the rhythm.
