# Idea Portfolio Template

Use this template to help the user create their initial `~/idea-portfolio.md`. Walk through each section together.

---

```markdown
# Idea Portfolio
**Owner**: [Name]
**Last full review**: [Date]
**Current phase**: Phase [1/2/3/4] — targeting $[X]K/month

---

## 🟢 Active Ideas

Ideas currently being worked on. Maximum 2 at a time (more than that = spreading too thin).

### [Idea Name]
**Status**: 🟢 Active
**Stage**: Phase [1/2/3]
**Started**: [date]
**Last updated**: [date]

**Original Conviction** (why we started this):
- [Reason 1]
- [Reason 2]

**Kill Criteria** (pre-agreed — if ANY of these are true, we stop):
- [ ] [Criteria 1 with specific number and deadline]
- [ ] [Criteria 2]
- [ ] [Criteria 3]

**Current Metrics**:
- Revenue: $[amount]/month
- Users/Customers: [count]
- Hours/week invested: [count]
- Trend: 📈/➡️/📉

**Compounding Score**: [1-5] — [reason it compounds with other work]

**Next Milestone**: [Specific milestone + deadline]

**Decision Log**:
- [date]: Started because [reason]

## 🟡 Validating

Ideas in validation phase — testing demand before committing real time.

### [Idea Name]
**Status**: 🟡 Validating
**Validation experiment**: [What are you testing?]
**Success criteria**: [What result means "go"?]
**Deadline**: [When do you decide?]
**Time budget**: [Max hours/week allocated to validation]

---

## 🔵 Parked Ideas

Ideas worth revisiting later, but not now. Include what would make them relevant again.

### [Idea Name]
**Parked on**: [date]
**Why parked**: [reason — timing, resources, prerequisite missing]
**Reactivation trigger**: [What would need to be true to pursue this?]
**Compounding score if active**: [1-5]

---

## 🔴 Graveyard

Killed ideas. Kept here so we don't revisit them by accident and to preserve the lessons.

### [Idea Name]
**Active from**: [start date] — [kill date]
**Original conviction**: [why it seemed good]
**What happened**: [what went wrong or changed]
**Kill criteria hit**: [which specific criteria triggered the kill]
**Lesson learned**: [what this taught us for future ideas]
**Time invested**: [total hours]
**Money invested**: [total spend]

---

## Weekly Review Checklist

Run this every Saturday during strategy time:

- [ ] Are active ideas still on track toward their next milestone?
- [ ] Has any idea hit its kill criteria? Be honest.
- [ ] Am I spending time proportional to each idea's potential?
- [ ] Is there a parked idea whose reactivation trigger has been met?
- [ ] Total hours/week invested: [number] — is this sustainable?
- [ ] Any shiny new objects to evaluate properly before acting on?
```

---

## Tips for the Initial Setup

When helping the user create their portfolio for the first time:

1. **Start with what's currently in flight.** Ask: "What are you actively spending time on right now, business-wise?"
2. **Dig for convictions.** For each idea, ask: "Why did you start this? What signal made you think this was worth your time?"
3. **Set kill criteria upfront.** This is the hardest part. Ask: "If you were advising a friend on this exact idea, what would you tell them are the warning signs to quit?"
4. **Cap active ideas at 2.** If they have 4 things going, 2 need to be parked or killed. Help them choose.
5. **Don't forget past kills.** Ask: "Any ideas you tried before that didn't work? Let's add them to the graveyard so we learn from them."
